What is Unit testing, pytest and how it works?


There are many types of automated testing out there: front-end testing, smoke testing,

load testing, end-to-end (E2E) testing, and that’s to name only a few.


Unit testing is one of the most valuable types of automated testing. Getting started

with it isn’t the easiest thing. Unit testing is one of the many different types of

automated testing. Unit tests exercise very small parts of the application in complete

isolation, comparing their actual behavior with the expected behavior.

The “complete isolation” part means you don’t typically connect your application with

external dependencies such as databases, the filesystem, or HTTP services. That allows

unit tests to be fast and more stable since they won’t fail due to problems with

those external services.


What is pytest?


First, pytest must be installed if it is not recognized as a command when you open

up python terminal after starting docker_flask application. I'll explain shortly

the configuration command "docker compose up --build" that you must use to start

docker_flask application.

I start by explaining what is pytest, its purposes and advantages.

Pytest is a Python testing framework that allows you to write code

to test if other code in working like it should. So that automated test

check run can test thousands of programs everytime you type "pytest"


If you a look at my Pycharm editor in the root folder there should be a

"requirements.txt file, that should have pytest inside of it.


My Pycharm editor:



If don't see pytest listed you would have to use a command prompt or window powershell

and type: "pip install -r requirements.txt" to install pytest module.


How to start docker_flask application.


  • In the project window root of the folders you should see a docker-compose.yml file, click on it to see the code.
  • First,s go to the bottom of pycharm and click on "Services" to start Docker container by pressing the green play button.
  • Go to the bottom of pycharm where you see "Terminal" click it.
  • When the Terminal opens up type: "docker compose up --build"

After you type this command you will see Container docker_flask created. The docker-compose.yml <- Orchestration file that contains the configuration to develop locally, it overrides the Dockerfile to run the flask development server instead of running the gunicorn server that is used for hosting on Heroku.
